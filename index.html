Basics of the Web

The Web -

A system of extensively interlinked hypertext documents. A branch of the Internet. Invented in 1990's. Over 30 billion pages.

HTML -

Hypertext Markup Language: The main type of document on the Web.

URL -

Uniform Resource Locator: a protocol for specifying addresses on the Internet. An address that identifies a particular file on the Internet, usually consisting of the protocol, as http, followed by the domain name.

HTTP -

Hypertext transfer protocol: the standard protocol for transferring hypertext documents on the World Wide Web.

Web Apps -

Any computer program that runs in a web browser. It is created in a browser-supported programming language (such as the combination of JavaScript, HTML and CSS) and relies on a web browser to render the application.

Major Pieces


You --- Computer (Browser) --- Internet (HTTP) --- Servers
HTML -

Hypertext Markup Languageis the standard markup language used to create web pages. It is written in the form of HTML elements consisting of tags enclosed in angle brackets (like <html>).
Text content= what you see. Markup= what it looks like; references to other documents: (e.g. images & videos); links to other pages. Element with tags = (opening tag -- contents -- closing tag)

Tag:

HTML tags are the hidden keywords within a web page that define how the browser must format and display the content. Most tags must have two parts, an opening and a closing part. For example, <html> is the opening tag and </html> is the closing tag.

Element:

An HTML element is an individual component of an HTML document or web page, once this has been parsed into the Document Object Model ("DOM"). HTML is composed of a tree of HTML elements and other nodes, such as text nodes. Each element can have HTML attributes specified.
It's everything within a set of opening/closing tags

Attribute:

An HTML attribute is a modifier of an HTML element. They are usually added to a HTML start tag, which along with content and end tag comprises most HTML elements. Particular attributes are only supported by particular element types; if they are added to elements that do not support them they will not function.
It's a property of an HTML element: <href>

HTML Attributes -

HTML elements can have attributes.

  • Attributes provide additional information about an element.
  • Attributes are always specified in the start tag.
  • Attributes come in name/value pairs like: name="value"
  • (attribute name - value - contents) Images - Whitespace - inline; block

HTML Block Elements and Inline Elements -

Most HTML elements are defined as block level elements or inline elements.
Block level elements normally start (and end) with a new line, when displayed in a browser.
Examples: <h1>, <p>, <ul>, <table>
Inline elements are normally displayed without line breaks.
Examples: <b>, <td>, <a>, <img>

span & div Elements -

A span element is in-line and usually used for a small chunk of HTML inside a line (such as inside a paragraph)
A div (division) element is block-line (which is basically equivalent to having a line-break before and after it) and used to group larger chunks of code.

Boxes

Box Sizing

There are four main points about box sizing.

  1. HTML elements are boxes and each box has 4 components.
  2. Because there are so many components to each box, it can often be hard to get the size of a box just right.
  3. There are two techniques you can use to help deal with sizing issues.
    a.Set sizes in terms of percentages rather than pixels.
    b.Set the box-sizing attribute to border-box for every element.
  4. Different browsers work slightly differently. Sometimes this causes different browsers to display the same code differently.
Box Positioning

Divs areblockelements (as opposed toinline), so by default they take up the entire width of a page.

Adding the ruledisplay: flex;to the appropriate CSS will override this behavior and let divs appear next to each other.

CSS

CSS is code written to control the "style" of HTML elements. CSS stands for "Cascading Style Sheets." The "C" is by far the most important part of this phrase. Cascading means "most specific" rules. Find the rule that best describes the elements. Then that rule gets applied.

Inheritance is a key feature in CSS, it relies on the ancestor-descendant relationship to operate. Inheritance is the mechanism by which properties are applied not only to a specified element, but also to its descendants. Inheritance prevents certain properties from being declared over and over again in a style sheet, allowing the software developers to write less CSS. It enhances faster-loading of web pages by users and enables the clients to save money on bandwidth and development costs.

In HTML, code comments begin with <!-- and end with -->. Everything in between is ignored. In CSS, code comments begin with /* and end with */.

Avoid Repetition

Programmers should always avoid repetition. Computers are not smart. They interpret instructions literally. When programmers repete themselves they create opportunites for human error and mistakes by potentially missing details in the program. The computer will take these missing details literally and create multiple problems for the programmer if they need to make changes.
